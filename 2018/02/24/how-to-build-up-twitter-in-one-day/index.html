How to build up "Twitter" in one day

Posted on 02-24-2018

This is a mini-project tutorial and demo for Web Development Application @CMU. A twitter-like social network platform has been built with Bootstrap 4 in Django 1.11. It enables user registration, user login, post, comment, following and updating profiles. Even though twitter has abundant API resources on its developer platform to exploit, it is still a good practice for new developers to achieve their cool ideas.

  1. Python 3.6

  2. Django 1.11

  3. Virtualenv (optional)

TemplateViews and FormViews provide much of functionality need to create a Django view. For instance, you can render many POST form back when receive a GET request.

Models

Create a model and model form to save posts and forms. The model form can just contain some fields that model has.

After the model created and changed, you need to make migrations, and migrate to your database.

If you delete some versions of migrations, but it comes to have errors when migrating, you probably need to do fake migrations. This is because you have inconsistent or conflicted fields in models and database. Fake migrations will apply those changes to database without running it, which can skip the error check.

User Authentication

So far we have created an anonymous-posting view. But if you want to have user authentication, you’ll need to introduce the functionality in Django to your services, or customize one.

Login

To save our efforts, Django user model has taken the authentication job for convenience and safety. You just need to add the login page and tell Django where it should direct after login. Same as in logout.

A HTML forked from a Bootstrap 4 example.

User Profiles

For every created user, you can generate a profile page for him/her to allow following and updating user’s info. Using a profile template is an easy approach to acheive that.

Models

First, we need a Profile model to extend default user model and save the info in database. Following are the descriptions of fields of a profile:

  • user : an One-to-One relationship field to an User instance. Retrieve the profile instance by user.user_profile.
  • img : enable users to upload their avatars. Default to be blank. Save the photo to media/ in MEDIA_ROOT in webapps/settings.py.
  • bio : allow users to write their self-introduction. Number of words limited. Default to be blank.
  • follower : a Many-to-Many relationship field that saves users the current user is following. Return all followers by user_profile.follower.all.

Settings

Update MEDIA_ROOT in webapps/settings.py

Add URLs and MEDIA_ROOT in socialnetwork/urls.py. Notice that it requires login to check users’ profiles, and each user will have its own profile URLs by username.

Views

  • GET : user authentication, render user info and profile form.
  • POST : user and form authentication, save profile. Profile form is not required to save but to retrieve the data, you have to save it with attribute commit=false first.
